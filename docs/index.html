02

How it works

jsdom + Three's SVGRenderer, baked into one optimized file.

STEP 01

Render frames

jsdom injects a DOM; Three's SVGRenderer turns each frame into vector SVG.

STEP 02

Flipbook

N frames are encoded into one SVG via one-hot opacity <animate> (SMIL) elements.

STEP 03

Commit

SVGO optimizes the result, and the Action commits it back into assets/ — like platane/snk.

03

Quickstart

Add a few lines to a workflow in your own repo. No need to vendor Three.js.

readme-art.yml# .github/workflows/readme-art.yml
- uses: kpab/three-readme@v1
  with:
    scene: crystal
    color: "#5cc8e8"
    out: assets/crystal.svg

- uses: stefanzweifel/git-auto-commit-action@v5
  with:
    file_pattern: "assets/*.svg"

Then embed it

Reference the generated file like any image: ![](./assets/crystal.svg). Because backgrounds are transparent, the same file sits cleanly on light and dark GitHub themes.

Want your name spinning in 3D? Use the text scene with --text "YOUR NAME".
